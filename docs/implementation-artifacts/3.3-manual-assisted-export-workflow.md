# Story 3.3: manual-assisted-export-workflow

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a User (Maya),
I want the tool to prepare everything for export and tell me exactly what to do,
so that I can get the NX Studio rendering with minimal friction.

## Acceptance Criteria

1. **Given** A batch of generated sidecars, **When** I run with `--export-guide` (or similar), **Then** The tool generates the sidecars (standard batch behavior).
2. **And** It automatically opens the output folder in the OS file explorer.
3. **And** It prints instructions: "Open NX Studio -> Select All in Folder X -> Export as JPEG".
4. **And** It waits (interactive prompt) for the user to confirm export completion.
5. **And** It validates the count of exported files (JPEGs/TIFFs) matches the input count of successfully processed NEFs.

## Dev Notes

- **Location:** Logic belongs primarily in the CLI layer (`cmd/nx`), wrapping the core `internal/batch` logic. The `orchestrator` should remain focused on Sidecar generation.
- **Project Structure:**
  - `cmd/nx/`: Command definitions and interactive flow.
  - `internal/batch/`: Core processing (unchanged, just called).
- **OS Support:** Go's `exec.Command` is sufficient. Ensure path sanitization.
- **Validation:**
  - NX Studio exports usually retain the filename (e.g., `DSC_1234.NEF` -> `DSC_1234.jpg`).
  - Match basenames ignoring extension to verify 1:1 correspondence.
- **Dependencies:** Standard library (`os/exec`, `path/filepath`, `bufio`) should suffice.
